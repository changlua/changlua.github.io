1、指定运行级别(init 数字)

运行级别介绍

设置运行级别：init 数字 /etc/inittab 这个目录存放着默认启动时的运行级别

运行级别说明：

​ 0 ：关机

​ 1 ：单用户【找回丢失密码】

​ 2：多用户状态没有网络服务

​ 3：多用户状态有网络服务

​ 4：系统未使用保留给用户

​ 5：图形界面

​ 6：系统重启

常用运行级别是 3 和 5 ，要修改默认的运行级别可改文件

示意图：

案例介绍

① 在线切换运行级别：

​ 输入 init 3,接着按下Enter

​

​ 接着我们输入用户名密码，注意密码尽量通过键盘F1、F2下的数字来输入，右边的可能无效，小键盘没打开

​

​ 我们切换回到图形界面，输入 init 5，Enter即可回到图形界面

​

② 查看默认运行级别：

我们前面已经说过 /etc/inittab 这个目录存放着默认启动时的运行级别，我们来查看一下

​ 输入命令：vim /etc/inittab查看

​

​ 能够看到，id:5 initdefault 默认级别为5

​

实际案例

① 如何找回root密码？

思路： 重启进入到 单用户模式，然后修改 root 密码。因为进入单用户模式，root 不需要密码就可以登录。

那么如何进入到单用户模式呢，那么我们就需要在系统启动过程中进行设置了：

具体步骤如下：

​ ① 我们首先重新启动，在启动中按下 alt，进入到如下界面

​

​ ② 我们按下 e ，来进行编辑，接着进入到这个界面，上下键锁定到第二行kernel…..

​

​ ③ 再输入 e ，接着输入空格和1(表示告诉内核我们要进入到单用户模式下)

​

​ ④ 我们回车之后，重新回到选择kernel…界面，输入 b(表示引导，重新启动)

​

​ ⑤ 接着我们输入 passwd root，来修改root用户的密码即可

​

最后来重新启动试一下是否能够更改密码吧！ 重新启动命令：reboot

注意：此时我们就会在想，如果是这样的话，那么每个知道这么操作的人不都可以进行更改root密码了嘛，安全性也太低了吧。实际上还是有个条件，只有你在本地机房的Linux上才能够这么操作，一般远程操作是不会给你在启动时进入到单击模式的！！！

② 如何设置系统默认运行级别？

解决思路：我们只需要到 /etc/inittab 这个文档中修改默认的运行级别即可在下次启动时进入到对应的运行级别。

在终端输入vim /etc/inittab ，来编辑其中的级别值保存退出即可！！！

若想重新改回来，重新启动后，依旧是更改下默认级别即可！

3、文件目录类

pwd指令(查看路径)

语法： pwd (显示当前工作目录的绝对路径)

ls指令(查看目录内)

语法：ls [选项] [目录或是文件] (查看当前目录的所有信息)

常用选项：

​ -a ：显示当前目录所有的文件和目录，包括隐藏的，如.开头文件。

​ -l ：以列表的方式显示信息。

或

​ -lh：以列表方式显示，并且能够按单位换算容量大小

cd指令(切换目录)

语法：cd [参数] (切换到指定目录)

常用参数：

​ 绝对路径 ：/home即表示从根目录开始定位 (如：cd /home)

​ 相对路径 ：../home及表示回到上层目录后定位 (如：cd ../home，根据情况来定) ../表示回到上级目录

​ ~或者无：回到自己家目录 (如: ：cd 或 cd ~)

实例案例：

起始路径为：/root/桌面

​ 案例一：使用绝对路径切换到 root 目录 cd /root

​ 案例二：使用相对路径到/home目录 cd ../../home (这里我们首先需要此刻在哪个位置再定位)

​ 案例三：表示回到当前目录的上一级目录 cd ../

​ 案例四：回到家目录 cd 或 cd ~

我们根据所在目录来选择对应合适的命令调整即可！！！

mkdir、rmdir指令(创建、移除)

对于使用以下命令时，也要看此时所在的目录，可以写绝对路径，也可以不写，依据情况来看

1、创建目录语法：mkdir [选项] (单级或多级创建目录)

​ 选项： -p # 创建多级目录

​

案例1：创建一个单级目录/home/dog mkdir /home/dog (/home目录已创建，dog目录没有创建)

案例2：创建一个多级目录/home/animal/dog mkdir -p /home/animal/dog

(/home目录已创建，/animal/dog 两个目录都没有创建，称为多级目录)

2、删除目录语法：rmdir 文件目录 (删除单级目录，且目录为空)

​ rm -rf 删除目录 (删除的目录下不为空)

案例1：删除上面的空目录/home/dog rm /home/dog (删除的是空目录)

案例2：删除上面创建的非空目录/home/animal rm -rf animal (删除的是非空目录)

touch指令(创建文档)

语法：touch 文件名称 (创建单个或多个空文件)

案例演示：创建一个空文件 touch cl.java 创建多个空文件 touch cl1.txt cl2.txt

cp指令(复制文档文件)

语法：cp [选项] source dest (拷贝文件到指定目录)

​ 选项：

​ -r：递归复制整个文件夹

实际案例：

​ 案例1：将多个文件复制到test目录下 cp cl.txt cl1.txt cl2.txt test

​ 案例2：将非空文件夹复制到另一个目录中 cp -r test/ test2/

​ 案例3：如何将文件复制到对应目录并强制覆盖 \cp -r test/ test2/

注意：要想要强制复制过去没有提示信息，那么开头就应该为\cp

rm指令(删除文档文件)

语法：rm [选项] # 要删除的文件或目录

​ 选项：

​ -r ：递归删除整个文件夹

​ -f ： 强制删除不提示

实际案例：

​ 案例1：删除单个文档 rm 1.txt

​ 案例2：删除非空文件 rm -rf test

​ 案例3：强制直接删除单个文档，没有提示 rm -f test

mv指令(重命名与移动)

语法：

​ mv oldNameFile newNameFile (功能描述：重命名)

​ mv /temp/movefile /targetFolder (功能描述：移动文件)

实例演示：

​ 案例1：重命名文档 mv 2.txt 1.txt 重命名文件 mv test3 test

​ 案例2：移动文档到目录 mv 1.txt test 移动非空目录到目录 mv test1 test

重命名的原理：例如 mv 1.txt 2.txt ，首先会执行移动操作，移动到对应目录中如果有1.txt的话，那么执行重命名更改为2.txt

cat、more、less(查看指令)

cat介绍：查看文件内容，是以只读的方式打开，只能浏览文件不能修改。

cat的语法：cat [选项] 要查看的文件

​ 选项：① -n ：显示行号 ② 可以带上管道命令 | more ，支持分页浏览

more介绍：是一个基于VI 编辑器的文本过滤器，它以全屏幕的方式按页显示文本文件的内容。

more的语法： more 要查看的文件

less介绍：用来分屏查看文件内容，它的功能与more 指令类似，但是比 more 指令更加强大，支持各种显示终端。less 指令在显示文件内容时，并不是一次将整个文件加载之后才显示，而是根据显示需要加载内容，对于显示大型文件具有较高的效率。

less的语法：less 要查看的文件

我们通过XFtp来将一个大型文件传到/opt的目录下，接着用三个指令来执行，并且可以使用快捷键：

​ cat -n 金庸-射雕英雄传txt精校版.txt | more # 空格、ctrl+f是向下一页，Enter向下一行

​ more 金庸-射雕英雄传txt精校版.txt # 空格、ctrl+f是向下一页 ctrl+b是向上一页 Enter向下一行

​ less 金庸-射雕英雄传txt精校版.txt # 向下键、Enter都是向下一行 向上键是向上一行

​ Pg Dn、ctrl+b是上一页 Pg Up、ctrl+f、空格是下一页

>指令与>>指令(覆盖与追加)

>指令：输出重定向：会将原来的内容覆盖

>>指令：追加，不会覆盖原来文件的内容，而是追加到文件的尾部

案例演示：

​ 案例1：列表的内容写入文件 a.txt 中（覆盖写） ls -l > a.txt

​ 案例2：列表的内容追加到文件 a.txt 的末尾 ls -l >> a.txt

​ 案例3：将/etc/profile文件的内容覆盖到c.txt cat /etc/profile > c.txt

​ 案例4：将指定内容追加到a.txt中 echo "hello world" > a.txt

​ 案例5：将/home 目录下的文件列表 写入到 /home/info.txt ls -l /home/ > /home/info.txt

​ 案例6：将当前日历信息 追加到 /home/mycal 文件中 cal >> /home/mycal

注意：需要说明的是，无论是覆盖还是追加，对应文件不存在时都会自动进行创建

echo指令(显示输出)

语法：echo [选项] [输出内容]

实际案例：

​ 案例1：使用 echo 指令输出环境变量,输出当前的环境路径 echo $PATH

​ 案例2：使用 echo 指令输出 hello,world! echo "hello,world!"

head、tail(显示开头结尾部分)

head介绍：用于显示文件的开头部分内容，默认情况下 head 指令显示文件的前 10 行内容

语法：head [选项] 文件

​ 选项：-n 5 查看头部5行内容

案例演示：查看/etc/profile 的前面 5 行代码 head -n 5 /etc/profile

tail介绍：用于输出文件中尾部的内容，默认情况下 tail 指令显示文件的后 10 行内容

语法：tail [选项] 文件

​ 选项：-n 5 查看尾部5条语句

​ -f 实时追踪该文档的所有更新，工作经常使用

案例演示：

​ 案例1：查看/etc/profile 最后 5 行的代码 tail -n 5 /etc/profile

​

​ 案例2：实时监控 mydate.txt , 看看到文件有变化时，是否看到实时的追加日期

​ ①首先远程输入命令 tail -f mydata.txt ，进入到实时监控中

​ ② 接着我们在虚拟机中的终端命令行中输入 ls -l >> /opt/mydata.txt，追加信息，能够在远程界面实时查看

​

in指令(软链接)

in指令描述：软链接也叫符号链接，类似于 windows 里的快捷方式，主要存放了链接其他文件的路径

基本语法：ln -s [原文件或目录][软链接名] (在当前目录添加一个软链接，进入该链接能够查看原文件内容)

案例演示：

​ 案例1：在/home 目录下创建一个软连接 linktoroot，连接到 /root 目录 ln -s /root linktoroot

​ 注意：使用pwd 查看路径依然是/home/linktoroot的原路径，ls能够查看到/root下的内容

​ 案例2：删除软链接linktoroot rm -rf linktoroot

​

​ 注意：在lintoroot后不要带有/，否则会提示资源忙

history指令(历史指令)

语法：history 功能描述：查看已经执行过历史命令

案例演示：

​ 案例 1: 显示所有的历史命令 history

​ 案例 2: 显示最近使用过的 10 个指令 history 10

​ 案例 3：执行历史编号为 393 的指令 !393

5、搜索查找类

find指令

介绍：find 指令将从指定目录向下递归地遍历其各个子目录，将满足条件的文件或者目录显示在终端。

查找相对于locate会慢，因为locate创建了对应的索引库

语法：find [搜索范围][选项]

选项：

​ -name：根据指定文件名查询

​ -user：查找属于指定用户名所有文件

​ -size：按照指定文件大小查找文件

实例演示：

​ 案例1：按文件名：根据名称查找/home 目录下的 hello.txt 文件 find /home -name hello.txt

​ 案例2：按拥有者：查找/opt 目录下，用户名称为 nobody 的文件 find /opt -user nobody

​ 案例3：查找整个 linux 系统下大于或小于或等于20m 的文件（+n 大于 -n 小于 n 等于）

​ 分别为： find / -size +20M find / -size -20M find / -size 20M

​ 案例4：查找所有文件名以.txt结尾的文件 find / -name *.txt 使用通配符来查找

注意：查询目录下文件列表以及对应文件的容量使用 ls -lh

locate指令(快速定位)

介绍：locate 指令可以快速定位文件路径。locate 指令利用事先建立的系统中所有文件名称及路径的locate 数据库实现快速定位给定的文件。Locate 指令无需遍历整个文件系统，查询速度较快。为了保证查询结果的准确度，管理员必须定期更新 locate 时刻。

注意第一次使用前需要创建locate数据库：updatedb

语法：locate 搜索文件名

实际案例：

​ 案例 1: 请使用 locate 指令快速定位 hello.txt 文件所在目录 locate hello.txt

管道符号 |和grep 指令

介绍：grep 过滤查找 ， 管道符，“|”，表示将前一个命令的处理结果输出传递给后面的命令处理。

grep写在管道符号|之后来作为过滤条件

基本语法：grep [选项] 查找内容 源文件

常用选项：

​ -n：显示匹配行及行号

​ -i：忽略字母大小写查找

实际案例：

​ 案例1：请在 hello.txt 文件中，查找 “yes”所在行，并且显示行号 cat hello.txt | grep -n yes

​ 案例2：基于案例1，查找不区分大小写的yes cat hello.txt | grep -ni yes

6、压缩和解压类

gzip、gunzip指令

gzip 用于压缩文件， gunzip用于解压的。很少使用

基本语法：

​ gzip 文件 （功能描述：压缩文件，只能将文件压缩为*.gz 文件）

​ gunzip 文件.gz （功能描述：解压缩文件命令）

实例演示：

​ 案例 1: gzip 压缩， 将 /home 下的 hello.txt 文件进行压缩 gzip hello.txt

​ 案例 2: gunzip 压缩， 将 /home 下的 hello.txt.gz 文件进行解压缩 gunzip hello.txt.gz

总结： 使用gzip压缩文件时，对应原文件不会在目录中存在。

​

zip、unzip指令

zip 用于压缩文件， unzip 用于解压的，这个在项目打包发布中很有用的。

基本语法：

​ zip [选项] XXX.zip 将要压缩的内容（功能描述：压缩文件和目录的命令）

​ unzip [选项] XXX.zip （功能描述：解压缩文件）

zip的选项：-r：递归压缩，即压缩目录

unzip的选项：-d<目录> ：指定解压后文件的存放目录

实例演示：

​ 案例1：将 /home 下的所有文件进行压缩成 mypackage.zip zip -r mypackage.zip /home/

​ 案例2：将mypackge.zip 解压到 /opt/tmp 目录下 unzip -d /opt/tmp/ mypackage.zip

​

注意：使用zip压缩时若是压缩/home到mypackage.zip，则会将整个home文件进行打包到mypackage.zip中；解压之后也就是home文件。

tar指令

可压缩可打包；tar 指令 是打包指令，最后打包后的文件是 .tar.gz 的文件。

基本语法：tar [选项] XXX.tar.gz 打包的内容 (功能描述：打包目录，压缩后的文件格式.tar.gz)

选项：

​ -c：产生.tar打包文件(打包)

​ -v：显示详细信息

​ -f：指定压缩后的文件名

​ -z：打包同时压缩

​ -x：解压.tar文件(解压)

实际案例：

案例1：压缩多个文件，将 /home/a1.txt 和 /home/a2.txt 压缩成 a.tar.gz tar -zcvf a.tar.gz a1.txt a2.txt

案例2：将/home 的文件夹 压缩成 myhome.tar.gz tar -zcvf myhome.tar.gz /home/

案例3：将myhome.tar.gzio解压到/opt/tmp目录下 tar -zxvf myhome.tar.gz -C /opt/tmp/

注意：① 在案例2中，依旧是将整个home目录打包为myhome.tar.gz压缩包，解压之后包含一个home文件夹。

​ ② 选项说明：压缩使用-zcvf；解压用-zxvf 并且后面还有个-C

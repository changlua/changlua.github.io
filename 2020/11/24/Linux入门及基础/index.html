1、Linux介绍

创始人：linux 林纳斯 吉祥物：企鹅tux

Linux：免费、开源、安全、高效、稳定，处理高并发非常强悍，需要企业级项目部署到Linux、unix服务器上运行。

主要发行版本(Linux内核) ：CentOSE、Redhat(自己公司做)、Ubuntu、Suse、红旗Llnux(中国做的)

Linux与Unix的关系？

关键人物：Ken tompson与Dennis richeres

关系图如下：

Ken tompson首先是在贝尔实验室工作，在参与分时系统项目后，由于没有起色，后发明了B语言，并用B语言开发了Unix，后因功能不强大。接着同Dennis richeres一起发明C语言(71年)，一起使用C语言进行重写并发布了Linux(73年)

后Richard Stallman发布了GNU计划，Linus Torvalds完成了Linux的第一次发布，Linux内核到现如今依旧在不断的由全世界开发人员在维护中：

4、vmtools的安装及使用

目的说明：

​ ① 在Windows与虚拟机中的CentOS能够相互复制粘贴

​ ② Windows与CentOS之间能够有共享文件夹

详细安装过程

首先点击vm上的虚拟机，选择安装vmware tools

此时我们虚拟机上出现了VMwareTools-10.3.10-139562.tar.gz，这个就是vmtools的压缩包

我们将这个压缩文件手动复制到/opt目录下(点击桌面-计算机-文件目录)，你可以手动右击直接解压，接着运行vmware-install.pl这个脚本文件，紧接着一直回车即可安装完成，安装完成之后，需要重启Centos才能够使用！！！

命令演示安装

cd /opt # 打开根目录下的opt文件

tar -zxvf VMwareTools-10.3.10-139562.tar.gz # 这个是解压命令，来对其中压缩文件解压

ll # 查看opt目录中的详细信息 其中vmware-tools-distrib就是解压目录

cd vmware-tools-distrib # 打开这个压缩文件

ls # 查看下该目录的所有文件

./vmware-install.pl # 安装这个.pl结尾的脚本文件

紧接着一直回车即可！

安装完后之后执行重启系统命令：reboot

重启完成之后，我们可以在windows与虚拟机中的centos进行相互复制粘贴了！！！

三、Linux的目录结构

基本介绍

Windows系统：分为多个盘符，盘符下有着各种文件

Linux：其文件系统的最上层为根目录，用/表示，采用层级式的树状目录结构

Linux世界里，一切皆为文件！

文件目录介绍

我们来查看一下Linux的文件目录分布：

下面是对应目录的说明及总结：

/mnt 【重点】:让用户挂载别的临时系统，如我们的共享文件夹
/sys、/proc、/srv 【重点】：这三个目录都与linux内核相关，千万不能乱动
/opt 给主机额外安装软件所存放地方，注意不是安装后的软件
/user/local 【重点】：另一个给主机额外安装软件所安装目录，一般通过编译源码方式安装，如安装mysql目录
/bin 【重点】:存放常用指令 如cat、cp、cd指令…
/etc 【重点】:存放配置文件,如my.conf
/home 【重点】：存放普通用户的目录，添加用户后，会产生对应的文件
/root 【重点】：root用户的文件及一些其他东西
/user 【重点】：用户的应用程序和文件都放这里，类似windows的program files目录
/var 【重点】：存放变量，不断变化的东西及日志
/dev： 设备管理器，硬件映射为文件来管理，如cpu、disk、console….
/lib： 动态库，.so结尾
/media： 能够识别插入的dvd、u盘等，识别之后挂载在这个目录
/sbin： 系统管理员能够使用的系统管理程序
/selinux (security-enhanced linux)：安全子系统，如出现了黑客攻击等，后进行触发，就像linux的360
/tmp： 存放临时文件夹
/boot： 存放启动linux命令的核心文件

本章Linux终端命令汇总

cd /文件名 # 打开根目录下的文件，只能单个单个目录打开

cd 文件名 # 如果在目录中，可以直接打开其中的文件

tar -zxvf 压缩文件 # 压缩现目录下文件

ll # 查看该目录的详细信息，包含总用量及所有文件名 与ls -1相同

ls # 仅仅查看该目录的下的文件详情

reboot # 重启系统

用户相关操作：

useradd 用户名 # 添加对应用户目录到/home目中

userdel -r 用户名 # 删除/home中的对应用户目录
